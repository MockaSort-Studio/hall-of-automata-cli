### Task 6: hall-init-board — link board to target repo after creation

**Files:**
- Modify: `skills/hall-init-board/SKILL.md:72-82` (Step 3, after project creation)

**Context:** `createProjectV2` creates the board at org/user level — it's a floating project not associated with any repo. GitHub Projects v2 supports `linkProjectV2ToRepository` to attach the project to a specific repo. Without this call, the board doesn't appear in the repo's Projects tab. The repo ID is fetched via GraphQL and is already available as `$OWNER` and `$REPO`.

- [ ] **Step 1: Add repo-linking mutation after project creation in Step 3**

After the line:

```bash
echo "Created Hall Board #${PROJECT_NUM} (${PROJECT_ID})"
```

Add (before the closing `PYEOF` or next bash block boundary — this is inside the Step 3 bash block):

```bash
REPO_NAME=$(echo "$REPO" | cut -d/ -f2)
REPO_ID=$(gh api graphql \
  -f query='query($o:String!,$r:String!){repository(owner:$o,name:$r){id}}' \
  -F o="$OWNER" -F r="$REPO_NAME" --jq '.data.repository.id')
gh api graphql \
  -f query='mutation($p:ID!,$r:ID!){linkProjectV2ToRepository(input:{projectId:$p,repositoryId:$r}){repository{name}}}' \
  -F p="$PROJECT_ID" -F r="$REPO_ID" --jq '.data.linkProjectV2ToRepository.repository.name'
echo "Board linked to repository ${REPO}."
```
